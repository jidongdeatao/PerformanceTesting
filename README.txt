# PerformanceTesting

  

  
  
性能测试流程：
学习业务->分析需求->工作评估->设计模型->编写计划->评审(通过)->准备测试环境、测试数据、开发好的脚本->Check(没有问题)->执行测试
过程中发现Bug要进行管理->分析性能汇报工作->准出检查（不满足的要进行系统调优)->编写报告汇报工作->End

判断执行是否通过的测试关注点：
  1.响应时间（RT）
  2.吞吐量（TPS）
  3.事务成功率
  4.硬件指标：CPU/内存/存储/网络
  5.稳定性
  6.内存有无泄漏
  7.其他（数据库、中间件、缓存、JVM）
  
优化技能：
  代码
  架构
  中间件
  操作系统
  数据库：配置、设计、SQL
